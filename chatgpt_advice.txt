**Core value proposition**
A continuously updated, quantitative “fame index” that ranks public figures across regions, sectors, and time. The hook is movement and comparison, not static fame.

**Data & methodology (critical)**

* Inputs: search volume (Google Trends), social mentions (X, Instagram, TikTok, Reddit), news volume, Wikipedia page views.
* Sentiment split: positive / negative / neutral, shown explicitly.
* Time weighting: recent spikes matter more than legacy fame.
* Transparency: publish a clear methodology page. This builds credibility and prevents “arbitrary score” criticism.

**Differentiation**

* “Momentum” score (week-on-week delta).
* Regional fame vs global fame comparison.
* Sector lenses (politics, sport, entertainment, tech).
* Controversy index (high fame + negative sentiment).

**User flows**

* Landing → Top 100 global → filter (region/sector) → individual profile.
* Profiles show trend graphs, sentiment mix, recent headlines.
* Internal links everywhere to maximise session depth.

**Content strategy**

* Weekly blog is smart:

  * “Biggest climbers this week”
  * “Fastest fallers”
  * “Why X spiked” (short, chart-driven posts)
* This content is SEO-friendly and shareable.

**Monetisation**

* Display ads initially.
* Later: sponsored analysis, data API, premium dashboards.
* Avoid paywalls early; scale traffic first.

**Risks**

* Data reliability and scraping limits.
* Legal caution: avoid defamatory framing; stick to “sentiment” not judgment.
* Algorithm gaming—expect attempts.

**Opinion**

This is a **canonicalisation failure**, and it will absolutely dilute indexing if not fixed decisively. The solution is mechanical.

### The rule (pick one, only one)

Either:

* `https://www.example.com` **or**
* `https://example.com`

There is no SEO advantage to either. Consistency is everything.

### Required fixes (all of them)

1. **301 redirect at server/CDN level**

* Non-preferred → preferred, globally.
* No exceptions, no mixed behaviour.
* This must happen before the page renders.

Examples:

* Nginx / Apache / CloudFront / Cloudflare — all can do this cleanly.

2. **Canonical tags**

* Every page must declare the preferred version:

```html
<link rel="canonical" href="https://www.example.com/path/" />
```

Never self-canon to the non-preferred variant.

3. **Internal links**

* 100% of internal links must use the preferred version.
* One stray link can reintroduce the problem at scale.

4. **Google Search Console**

* Add **both** properties:

  * `https://example.com`
  * `https://www.example.com`
* Set the preferred one.
* Submit sitemaps **only** for the preferred version.

5. **Sitemaps**

* URLs must all match the preferred domain.
* No mixed entries. Ever.

6. **External tools / CMS**

* Check CMS base URL, analytics, tag manager, feeds, APIs.
* These often leak the wrong variant.

### Common mistakes to avoid

* Using canonicals without redirects (Google treats this as a hint, not a rule).
* Letting Cloudflare/CDN handle redirects inconsistently.
* Mixing www/non-www between desktop and mobile.

No. Google does **not** prefer non-[www](http://www). It treats `www` and non-`www` as **separate hosts** and will index whichever you make authoritative.

Your problem is not Google’s preference; it is **your site signalling both**.

### Facts (not opinions)

* Google is neutral on `www` vs non-`www`.
* Google indexes what is:

  * consistently redirected,
  * consistently canonicalised,
  * consistently linked.
* Mixed signals = split indexing (what you are seeing).

### Which should you choose?

From a practical standpoint:

* **Non-www is simpler** and common for modern sites.
* **www is technically cleaner** if you ever need subdomain routing, cookies, or CDN tricks.

**SEO impact: zero difference.**

### What matters more than the choice

Once you choose, you must enforce it absolutely:

* One canonical host
* One sitemap host
* One internal-link host
* One redirect rule

### Recommendation

If you are already seeing Google index non-www more:

* Standardise on **non-www**
* Force **301 www → non-www**
* Fix canonicals and sitemaps to match
